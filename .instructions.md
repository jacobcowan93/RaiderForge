# RaiderForge Auth & Deploy Checklist

Always develop and test against `http://localhost:3000` first. Never build raw Discord/Google OAuth URLs; always call `signIn('discord')` or `signIn('google')`.

## Local env (.env.local)

```
NEXTAUTH_URL=http://localhost:3000
NEXTAUTH_SECRET=<local secret - generate via: openssl rand -base64 32>
DISCORD_CLIENT_ID=<from Discord portal>
DISCORD_CLIENT_SECRET=<from Discord portal>
GOOGLE_CLIENT_ID=<from Google Cloud>
GOOGLE_CLIENT_SECRET=<from Google Cloud>
```

## Discord Redirects

Add these **plain text** URLs to Discord Developer Portal → OAuth2 → Redirects:

```
http://localhost:3000/api/auth/callback/discord
https://raiderforge.org/api/auth/callback/discord
```

(Remove any redirect that is just `https://www.raiderforge.org/` or similar—it must be the full callback path.)

## Google Redirects

In Google Cloud Console → APIs & Services → Credentials → your OAuth client:

**Authorized JavaScript origins:**
```
http://localhost:3000
https://raiderforge.org
```

**Authorized redirect URIs:**
```
http://localhost:3000/api/auth/callback/google
https://raiderforge.org/api/auth/callback/google
```

## Vercel (Production env)

In Vercel dashboard → RaiderForge → Settings → Environment Variables → Production:

```
NEXTAUTH_URL=https://raiderforge.org
NEXTAUTH_SECRET=<prod secret - different from local, generate via: openssl rand -base64 32>
DISCORD_CLIENT_ID=<same as local>
DISCORD_CLIENT_SECRET=<same as local>
GOOGLE_CLIENT_ID=<same as local>
GOOGLE_CLIENT_SECRET=<same as local>
```

Then redeploy the `main` branch.

## Workflow

1. Fill `.env.local`, run `npm run dev`, test both providers on `http://localhost:3000`.
2. Commit + push to `main`.
3. After Vercel deploy, test both providers on `https://raiderforge.org`.

## Key Technical Notes

- **Dev**: `http://localhost:3000`
- **Production**: `https://raiderforge.org`
